🔬 Multi-Layer Image Quality Engine

Every uploaded image goes through a two-layer analysis before acceptance:

Layer 1 — Pixel Analysis (Sharp)

Laplacian Variance: Computes edge variance to detect blur.
Sobel Edge Density: Edge density as secondary confirmation.
Pixel Statistics: Mean brightness, standard deviation for contrast.
Grid Glare Detection: Image split into 8×8 grid — each cell checked for brightness, color saturation, and edge density to detect flash reflections.

Layer 2 — OCR Verification (Tesseract)

OCR Confidence Score: Text extracted from image — if confidence > 65%, pixel-level warnings are dismissed as false positives.
OCR Override: If score < 50% with sufficient words found, image is flagged as blurry regardless of pixel analysis.
Language Support: English + Malay (eng+msa).

Draft — saved but not submitted  →  Pending — submitted, waiting for admin  →  Taken — claimed by admin for processing  →  Released — returned to pending queue

Atomic Case Assignment

Admins claim cases for processing. Uses updateMany with condition taken_by = null — ensures no two admins can claim the same case simultaneously (race-condition proof).

Case Release

Cases can be released back to the pending queue. Release reason is recorded in release_reason. Release date and take date are tracked separately.

Advanced Filtering

"Taken Cases" list supports: text search (name, IC, employer), date filtering (this month, last month, 3 months, this year, custom range), filter by taker, and pagination (20 per page).

Internal Notes

An agent_message field for internal communication between agents and admins. Every note update is broadcast in real-time via WebSocket.

Admin Case Files

Admins can upload additional files (letters, notes) with custom labels to any case. These files are stored separately from customer files in the admin_files/ folder.

Encrypted Search

Since applicant data is encrypted (AES-256), search is performed client-side after decryption — ensuring data privacy even during query operations.

Module 5 — Automated PDF Generation Engine

Smart field mapping engine with AI Vision support and multi-row text filling

▼

PDF Template Upload

Upload official loan form PDFs (up to 50MB) — the system automatically reads all interactive form fields. Supports single-file products and multi-file products (parent with sub-documents).

Visual Field Mapping

Visual mapping interface — admins select PDF fields and map them to one of 30 standard data fields. Actual PDF preview displayed side-by-side for reference.

🤖 AI Auto-Mapping (Gemini Vision)

System captures screenshots of each PDF page, sends to Gemini 2.5 Pro with field list and positions — AI suggests mappings automatically. Supports section-by-section processing for high accuracy.

Smart Multi-Row Text Filling

When a long address is mapped to 3 PDF rows, text is automatically split at word boundaries (~50 characters per row). Adjacent comb/boxed fields are chained into a single text stream.

Automatic Calculations from IC Number

Date of Birth

Extracted from first 6 IC digits (YYMMDD). Years 00-29 → 2000s; 30-99 → 1900s. Output format: DD/MM/YYYY.

Gender & Age

Last IC digit: odd = Male (L), even = Female (P). Age calculated from date of birth to current date.

Product Hierarchy

Products can have sub-products (children). Example: "Personal Loan" can contain "Form A", "Form B", "Agreement". Each sub-product has its own field mapping.

Module 10 — Security & Data Protection

Field-level encryption, secure authentication, and multi-layer access control

▼
🔐

AES-256-CBC Encryption

All applicant personal data (name, IC, address, employment info, references) is encrypted before storage in the database. Encryption key derived via crypto.scryptSync with salt. Unique IV generated for every encryption operation.

🔑

Password Hashing

User passwords are hashed using bcryptjs with 10 rounds. The original password is never stored. Verification uses bcrypt.compare().

🛡

Persistent Session Store (MySQL)

Sessions are stored in a MySQL table (not in-memory) — they survive server restarts. 24-hour session cookie duration. Proxy trust enabled for Cloudflare Tunnel deployments.

🤖

Search Engine Blocking

Header X-Robots-Tag: noindex, nofollow is set on every response — prevents search engines from indexing any page of the system.
